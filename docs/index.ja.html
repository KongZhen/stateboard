プロダクト概要

散らばった文脈を、行動できるシグナルへ。更新され続けるプロジェクトボード。

Stateboard は AI の会話、プロジェクトファイル、ノート、TODO、検証結果を、継続的に更新されるプロジェクト状態ボードに変換します。作業の途中で証拠を残し、次のアクションを見える状態に保ち、境界が明確なときだけ適切なツールへ同期します。

Stateboard プロジェクト · GitHub Pages · alpha

  • TODO リストではありません。何が起きたか、なぜ重要か、次に何を進めるかを記録します。
  • 証拠が先です。各行はドキュメント、ノート、出力、検証結果へたどれる必要があります。
  • 同期には境界があります。宛先、階層、タイミングが明確なときだけ外部システムへ書き込みます。

「目的は tracking を増やすことではなく、説明し直す回数を減らすことです。」

01

TODO assistant ではない

Stateboard は TODO リストやタスク管理ツールではありません。作業コンテキストを、何が起きたか、なぜ重要か、次に何をするかへ要約する状態レイヤーです。

  • 意思決定、出力、リンク、学びを保持します。
  • 本当に必要な次のアクションを浮かび上がらせます。
  • 履歴を見える、検索できる状態に保ちます。
02

証拠ワークフロー

証拠が土台です。すべての更新は、開ける note、log、output、link に裏づけられるべきです。

  • 作業しながらコンテキストを捕捉します。
  • artifact をリポジトリへリンクします。
  • 同期境界で見直します。
03

クイックスタート

サービスをインストールしなくても、リポジトリやスレッドでボードを作れます。

  1. プロンプトをコピーします。
  2. AI assistant で実行します。
  3. 生成された board を保存するか、確認済みの宛先へ同期します。
04

連携階層

連携は幅ではなく、深さと信頼性で説明します。

  • 現在: プロンプトと Codex スキル。
  • 設定済み: 実 connector があるときだけ書き込み。
  • Fallback: Markdown または export rows。
クイックスタート: プロンプトをコピー ひとつのプロンプト。どの assistant でも。

以下のプロジェクトまたは会話コンテキストをプロジェクト状態ボードに整理し、指定した宛先へ同期してください。

整理したい内容:
[例: 提供できる、または読める Codex / ChatGPT / Claude / DeepSeek / WorkBuddy の直近 60 日分の会話、現在のプロジェクト、プロジェクトフォルダ、既存の TODO、貼り付けた複数のプロジェクト会話]

同期先:
[例: Dida/TickTick、Notion、Google Calendar、Feishu/Lark、DingTalk、WeCom、Markdown のみ]

対象場所:
[例: Dida/TickTick の「Project Control」リスト、Notion のページ/データベース、新しい XXX リスト、Markdown のみなら「このスレッドに返信」]

自動更新:
[不要 / 毎日 HH:MM / 毎週 曜日 HH:MM。明確な時刻がない場合は「不要」と書く]

ルール:
- 読める、検索できる、または私が提供した内容だけを使う。
- まず証拠からプロジェクト状態を判断する。チャット履歴をそのまま大量のタスクに分割しない。
- 既定では、実プロジェクトごとに 1 つのプロジェクト状態タスクだけを作成または更新する。
- 書き込み前に既存タスクを検索する。同じプロジェクトがあれば更新する。
- 自動更新に頻度と正確な時刻がない場合は、先に確認する。既定時刻を仮定しない。
- token、webhook secret、license key、private credential、private destination identifier を書き込まない。
- 完了後、作成、更新、検証、まだ確認が必要なことを報告する。

連携階層

広さより深さ。

Stateboard は support tier なしに「すべてをサポート」とは言いません。タスク同期、カレンダー同期、通知配信、Markdown fallback は別の約束です。

  • 現在の入口はプロンプトと Codex スキルです。
  • 実書き込みには connector discovery と検証が必要です。
  • 自動化は頻度と正確な時刻が確認されるまで待ちます。
