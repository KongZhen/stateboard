제품 개요

흩어진 컨텍스트를 실행 가능한 신호로. 계속 갱신되는 프로젝트 보드.

Stateboard는 AI 대화, 프로젝트 파일, 노트, 할 일, 검증 결과를 계속 업데이트되는 프로젝트 상태 보드로 정리합니다. 작업 중 생기는 근거를 남기고, 다음 행동을 보이게 하며, 경계가 명확할 때만 적절한 도구로 동기화합니다.

Stateboard 프로젝트 · GitHub Pages · alpha

  • 할 일 목록이 아닙니다. 무엇이 일어났고, 왜 중요한지, 다음에 무엇을 해야 하는지 기록합니다.
  • 근거가 먼저입니다. 각 행은 문서, 노트, 출력, 검증 결과로 추적 가능해야 합니다.
  • 동기화에는 경계가 있습니다. 대상, 계층, 시간이 명확할 때만 외부 시스템에 씁니다.

“목표는 더 많은 tracking 이 아니라, 다시 설명하는 일을 줄이는 것입니다.”

계획된 어댑터

어댑터는 계층별로 진행됩니다. 아래 연동 계층에서 기대 범위를 확인하세요.

계층 보기 ↓
01

TODO assistant 가 아닙니다

Stateboard 는 할 일 목록이나 태스크 관리자가 아닙니다. 작업 컨텍스트를 무엇이 일어났고, 왜 중요하고, 다음에 무엇을 해야 하는지로 요약하는 상태 레이어입니다.

  • 결정, 출력, 링크, 학습을 보존합니다.
  • 진짜 필요한 다음 행동을 드러냅니다.
  • 히스토리를 보이고 검색 가능하게 유지합니다.
02

근거 워크플로

근거가 기반입니다. 모든 업데이트는 열어볼 수 있는 note, log, output, link 로 뒷받침되어야 합니다.

  • 작업하면서 컨텍스트를 캡처합니다.
  • artifact 를 repo 에 연결합니다.
  • 동기화 경계에서 검토합니다.
03

빠른 시작

서비스를 설치하지 않고도 repo 또는 스레드에서 보드를 만들 수 있습니다.

  1. 프롬프트를 복사합니다.
  2. AI assistant 에서 실행합니다.
  3. 생성된 board 를 저장하거나 확인된 대상으로 동기화합니다.
04

연동 계층

연동은 플랫폼 수가 아니라 깊이와 신뢰성으로 설명합니다.

  • 현재: 프롬프트와 스킬.
  • 설정됨: 실제 connector 가 있을 때만 쓰기.
  • Fallback: Markdown 또는 export rows.
빠른 시작: 프롬프트 복사 하나의 프롬프트. 어떤 assistant 에서도.
prompts/stateboard-sync-prompt.ko.md 
다음 프로젝트 또는 대화 컨텍스트를 프로젝트 상태 보드로 정리한 뒤, 내가 지정한 대상으로 동기화해 주세요.

정리할 내용:
[예: 내가 제공할 수 있거나 읽을 수 있는 Codex / ChatGPT / Claude / DeepSeek / WorkBuddy의 최근 60일 대화, 현재 프로젝트, 프로젝트 폴더, 기존 TODO, 붙여 넣은 여러 프로젝트 대화]

동기화 대상:
[예: Dida/TickTick, Notion, Google Calendar, Feishu/Lark, DingTalk, WeCom, Markdown만 출력]

대상 위치:
[예: Dida/TickTick의 "Project Control" 목록, Notion 페이지/데이터베이스, "XXX라는 새 목록 만들기", Markdown만이면 "이 스레드에 바로 답변"]

자동 업데이트:
[필요 없음 / 매일 HH:MM / 매주 요일 HH:MM. 명확한 시간이 없으면 "필요 없음"이라고 쓰기]

규칙:
- 읽거나 검색할 수 있거나 내가 제공한 내용만 사용한다. 보이지 않는 대화나 프로젝트 자료가 있으면 무엇이 부족한지 알려 준다.
- 먼저 근거로 프로젝트 상태를 판단한다. 대화 기록을 그대로 많은 작업으로 쪼개지 않는다.
- 기본적으로 실제 프로젝트마다 프로젝트 상태 작업 1개만 생성하거나 업데이트한다. 현재 진행 상황, 위험/블로커, 다음 단계, 근거를 포함한다.
- 각 작업에는 정말 유용한 다음 단계 3-5개만 남긴다.
- 쓰기 전에 기존 작업을 검색한다. 같은 프로젝트의 상태 작업이 이미 있으면 중복 생성하지 말고 업데이트한다.
- 현재 환경에서 대상 도구에 쓸 수 없으면, 복사하기 쉬운 Markdown 버전을 출력하고 블로커를 설명한다.
- 자동 업데이트를 요청했지만 빈도와 정확한 시간이 모두 없으면 먼저 질문한다. 기본 시간을 가정하지 않는다. 현재 환경에서 자동화를 만들 수 없으면 만들었다고 주장하지 말고 복사 가능한 설정 안내를 제공한다.
- token, webhook secret, license key, private credential, private destination identifier를 쓰지 않는다.
- 내가 명시적으로 이중 언어 출력을 요청하지 않는 한, 내 요청 언어로 답하고 같은 언어로 대상 내용을 작성한다.
- 완료 후 무엇을 만들었는지, 무엇을 업데이트했는지, 어떻게 검증했는지, 아직 확인이 필요한 것이 무엇인지 알려 준다.

연동 계층

폭보다 깊이.

Stateboard 는 support tier 없이 “모두 지원”이라고 말하지 않습니다. 태스크 동기화, 캘린더 동기화, 알림 전달, Markdown fallback 은 서로 다른 약속입니다.

  • 현재 진입점은 프롬프트와 스킬입니다.
  • 실제 쓰기는 connector discovery 와 검증이 필요합니다.
  • 자동화는 빈도와 정확한 시간이 확인될 때까지 기다립니다.
