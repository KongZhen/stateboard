# Stateboard 同期プロンプト

[English](stateboard-sync-prompt.md) · [中文](stateboard-sync-prompt.zh.md)

これは Stateboard のゼロインストール入口です。下のプロンプトをコピーし、角括弧内を置き換えて使います。

```text
以下のプロジェクトまたは会話コンテキストをプロジェクト状態ボードに整理し、指定した宛先へ同期してください。

整理したい内容:
[例: 提供できる、または読める Codex / ChatGPT / Claude / DeepSeek / WorkBuddy の直近 60 日分の会話、現在のプロジェクト、プロジェクトフォルダ、既存の TODO、貼り付けた複数のプロジェクト会話]

同期先:
[例: Dida/TickTick、Notion、Google Calendar、Feishu/Lark、DingTalk、WeCom、Markdown のみ]

対象場所:
[例: Dida/TickTick の「Project Control」リスト、Notion のページ/データベース、新しい XXX リスト、Markdown のみなら「このスレッドに返信」]

自動更新:
[不要 / 毎日 HH:MM / 毎週 曜日 HH:MM。明確な時刻がない場合は「不要」と書く]

ルール:
- 読める、検索できる、または私が提供した内容だけを使う。見えない会話やプロジェクト資料がある場合は、何が不足しているかを伝える。
- まず証拠からプロジェクト状態を判断する。チャット履歴をそのまま大量のタスクに分割しない。
- 既定では、実プロジェクトごとに 1 つのプロジェクト状態タスクだけを作成または更新する。現在の進捗、リスク/ブロッカー、次のステップ、証拠を含める。
- 各タスクの次のステップは、本当に有用な 3-5 個だけに絞る。
- 書き込み前に既存タスクを検索する。同じプロジェクトの状態タスクがあれば、重複作成せず更新する。
- 現在の環境から対象ツールへ書き込めない場合は、コピーしやすい Markdown 版を出力し、ブロッカーを説明する。
- 自動更新を希望していても、頻度と正確な時刻が両方ない場合は先に確認する。既定時刻を仮定しない。現在の環境で自動化を作成できない場合は、作成済みと主張せず、コピーできる設定手順を出す。
- token、webhook secret、license key、private credential、private destination identifier を書き込まない。
- 明示的にバイリンガル出力を依頼しない限り、私の依頼の言語で返答し、同じ言語で宛先内容を書く。
- 完了後、作成したもの、更新したもの、検証方法、まだ確認が必要なことを報告する。
```

## 最小テスト入力

```text
整理したい内容:
現在の Stateboard プロジェクト、または <your-project-path> のプロジェクト

同期先:
Dida/TickTick

対象場所:
Project Status Test という新しいリストを作成

自動更新:
不要
```
