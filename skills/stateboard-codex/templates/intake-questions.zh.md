# Intake Questions

[English](intake-questions.md)

创建任务或自动化前使用这些问题。发送给用户前，按用户请求语言调整。

## Minimal Intake

```text
我先确认 5 个设置，然后再写入待办系统：

1. 你想整理哪些内容？例如近 60 天能提供或我能读取的 AI 对话、当前项目、某个项目文件夹、已有待办，或几段粘贴的聊天记录。
2. 你想同步到哪里？例如滴答清单、Notion、Google Calendar、飞书、钉钉、企业微信，或只输出 Markdown。
3. 目标位置叫什么？例如某个清单、页面、数据库、日历；没有的话我可以新建。
4. 是否需要自动更新？不需要、每天某个时间、或每周某个时间。
5. 如果你有偏好：要紧凑总览、平衡项目任务，还是详细行动清单？不说的话我默认用平衡项目任务。
```

## Time Confirmation

```text
你希望自动化在什么时间运行？

常见选择：
- 07:30：开始工作前看项目状态
- 09:00：工作日启动时整理
- 18:30：下班前回收当天状态
- 22:30：晚上复盘后更新
- 每周一 09:00：做周计划，不做每日更新
```

## Granularity Confirmation

```text
任务颗粒度我建议默认用「平衡项目任务」：

- 每个活跃项目 1 条任务
- 每条任务 3-5 个下一步
- 低频项目收进 parking lot

如果你想要更轻，可以只做总览；如果你准备执行某个项目，可以对那个项目展开成详细行动清单。
```
